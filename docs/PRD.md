# Notarie — Notion 기반 개인 블로그 MVP PRD

## 🎯 핵심 정보

**목적**: 별도 어드민 없이 Notion 앱에서 글을 작성·발행하면, 방문자가 읽기 전용으로 소비하는 정적 개인 기술/일상 블로그를 제공한다.
**사용자**: 검색·링크를 통해 유입되어 글을 읽기만 하는 익명 독자 (개발자·기술 관심 일반 독자).

## 🚶 사용자 여정

```
1. 홈 (최신 글 목록)
   ↓ 글 카드 클릭
2. 글 상세
   ↓ 본문 내 태그/카테고리 클릭 (분기)
   [태그 클릭]     → 태그별 글 목록
   [카테고리 클릭] → 카테고리별 글 목록
   ↓ 목록에서 다른 글 선택
   → 글 상세 (반복)

별도 진입 흐름:
- 홈/모든 페이지 상단 메뉴
   ↓ "검색" 클릭
   검색 → 키워드 입력 → 결과 글 선택 → 글 상세
   ↓ "소개" 클릭
   소개 (About)

최종 상태: 원하는 글을 끝까지 읽거나, 관련 글(태그·카테고리)로 탐색을 이어감.
```

## 🗄️ 데이터 모델

> 콘텐츠 저장소는 **Notion 데이터 소스**이며, 별도 RDB(Supabase 등)는 **불필요**합니다. 아래는 Notion 포스트 스키마를 데이터 모델로 표현한 것입니다. API는 `2025-09-03` 버전의 **data source 모델**을 사용하여 `data_source_id` 기준으로 쿼리합니다.

### Post (블로그 글 — Notion 데이터 소스)

| 필드        | 설명                                      | 타입/관계            |
| ----------- | ----------------------------------------- | -------------------- |
| id          | Notion 페이지 ID                          | string (page id)     |
| title       | 글 제목                                   | title                |
| slug        | URL 경로 식별자                           | rich_text            |
| status      | 발행 상태 (Draft 비노출 / Published 노출) | status               |
| publishedAt | 발행일 (목록 정렬 기준)                   | date                 |
| tags        | 태그 목록                                 | multi_select         |
| category    | 카테고리                                  | select               |
| excerpt     | 요약 (목록·SEO description)               | rich_text            |
| cover       | 커버 이미지                               | files / 페이지 cover |
| author      | 작성자                                    | rich_text / people   |

### PostContent (본문 — 파생 데이터, 저장 안 함)

| 필드     | 설명                                 | 타입/관계                   |
| -------- | ------------------------------------ | --------------------------- |
| blocks   | 페이지 블록 트리 (Notion API로 조회) | block[]                     |
| markdown | 블록 변환 결과 (렌더용 중간 산출물)  | string                      |
| images   | 영구화 대상 이미지 URL 매핑          | { originUrl → storedUrl }[] |

## 🛠️ 기술 스택 (최신 버전)

**프론트엔드**

- Next.js 15 (App Router) + ISR
- React 19
- TypeScript 5

**스타일링 · UI**

- TailwindCSS v4
- shadcn/ui (new-york)
- Lucide React (아이콘)

**콘텐츠 소스 (Notion)**

- `@notionhq/client` v5 (`2025-09-03`, data source 모델 / `data_source_id` 쿼리)
- 별도 데이터베이스 없음 — Notion이 단일 콘텐츠 백엔드 (선택: 검색 가속이 필요할 때만 외부 인덱스 고려)

**렌더링**

- `notion-to-md`(또는 동등 변환 라이브러리)로 블록 → 마크다운 변환
- 마크다운 렌더러 + 코드 신택스 하이라이팅 (예: Shiki/rehype-pretty-code)
- 이미지 영구화: 빌드 시 다운로드 또는 프록시 처리

**배포**

- Vercel (SSG + ISR, 시간 기반/수동 revalidate)

**패키지 관리**

- npm
